Demo: Bootstrapping the Viking Blogger Homepage

In this demo, we'll walk through rebuilding the Viking Blogger homepage using Bootstrap, HTML5, and SASS. We already created this homepage using standard CSS in the CSS demo. This time, we'll apply the Bootstrap framework and talk about some of the pros and cons of using it in your own projects.

We'll also start using the SASS preprocessor so we can nest our CSS styles inside each other. Check out the lesson on SASS if you need a refresher. This is the same preprocessor used by Rails, so you'll find many of these patterns familiar (and easier) once we start using that framework.

This demo will focus on the technical aspects of building using Bootstrap, HTML5, and SASS. We won't be covering the process side of things like we have in other lessons (e.g. using Pivotal Tracker stories). At the end, we'll take a look at some of the pros and cons of using Bootstrap based on how it all came together.

This demo is text-based so you can come back and more easily pinpoint questions.

Note: Files are all available in the Github Repository

2. Setup Basic File Structure

We've opened up index.html and set up the basic doctype structure for a "Hello World" page, just to make sure everything is loading properly. We've included Bootstrap's CSS and Javascript files as well as jQuery. We also skipped ahead a bit and included our custom CSS file:

Note that we don't need a separate reset.css file here because Bootstrap takes care of that for us.

Our Custom CSS file

We need to create our own custom CSS styles file which will layer on top of Boostrap's default styles. We'll use the .scss extension so SASS will precompile it.

We'll import the standard .css extension version (not the .scss one) since that's what SASS will convert the .scss file to:

...
<link href="styles.css" rel="stylesheet">
...

Set up SASS

To get SASS automatically listening for changes to our .scss files and compiling them into the .css versions, simply tell SASS to watch your current directory:

# If you haven't already
$ gem install sass

# Watch the current directory for changes
# This will require a dedicated terminal session (use a new tab)
$ sass --watch .
#>>> Sass is watching for changes. Press Ctrl-C to stop.

Tell Git to Ignore the .sass-cache Folder

Now the last thing we need to do is ignore the .sass-cache directory which SASS creates in our home directory. We don't want that getting sucked up to Github because it's not part of our core repo and there's no reason for it to be up there. You should get in the practice of "pruning" unwanted files and directories from being included in Git.

Do this by creating a special dotfile called .gitignore in your project root directory. Include the extensions or names of any files you don't want Git to recognize and Git will ignore them completely. Comments are denoted by the pound #.

You'll see this again in Rails:

# /.gitignore

# Ignore the SASS cache
.sass-cache

Now, if we do a $ git status, you can see that the .sass-cache is nowhere to be found, as desired:

$ git status
# On branch master
# Your branch is up-to-date with 'origin/master'.

# Untracked files:
#   (use "git add <file>..." to include in what will be committed)
# 
#     .gitignore
#     index.html
#     styles.css
#     styles.scss

4. Building the Navbar

Now, in the absence of any priority which would normally be provided by user stories, we'll start at the top of the page and work our way down. That means beginning with the navbar.

This is where we will introduce Bootstrap. It's worth noting that we're going to try to keep a light touch when applying Bootstrap's special classes. We'll focus on those that overcome some of the more annoying aspects of coding raw CSS like grid-friendly containers, dropdown menus, and buttons.

We also introduce SASS's CSS nesting structure for the first time.

Quick Note on Navbars in Bootstrap

Bootstrap navbars are actually some of their most complicated components because there are so many moving pieces. You can find their example CSS on the Components page of their docs. Because of this complexity, we'll essentially copy-paste their example markup to form the base of our navbar and then leave behind the parts we don't need.

Some of the markup they include in their example may seem irrelevant but is actually used for their responsive (mobile-friendly) features. Specifically, this refers to the part that looks like:

...
<button type="button" 
        class="navbar-toggle collapsed" 
        data-toggle="collapse" 
        data-target="#bs-example-navbar-collapse-1">
  <span class="sr-only">Toggle navigation</span>
  <span class="icon-bar"></span>
  <span class="icon-bar"></span>
  <span class="icon-bar"></span>
</button>
...

That section is used to create the little pressable icon you find on the right side of collapsed navbars on mobile browsers. To see it in action, go back to the docs and reduce the screen size while watching the example navbar they provide.

Building the Navbar

  1. We start by tacking on Bootstrap's special navbar classes to our <nav> element:

    ...
<nav id="topnav" class="navbar navbar-default">
...

  2. For our logo and site tile (the .logo-container section), we just make sure to add the special navbar-brand class to the <a> link with our "Viking Blogger" name so it is properly centered and displayed.

    <div class="logo-container">
  <img class="pull-left" src="http://seeklogo.com/images/V/v-tele-logo-053AACAF21-seeklogo.com.gif">
  <a class="navbar-brand" href="#">
    Viking Blogger
  </a>
  ...
</div>

  3. We then set up the right hand buttons and dropdown as list items wrapped in several Bootstrap classes:

    ...
<ul class="nav navbar-nav navbar-right nav-links">
...

  4. The "Create Post" button is just a list item with its own class:

    ...
<li class="highlighted-btn">
  <a href="#">
    Create Post
  </a>
</li>
...

  5. We used a Bootstrap hex color code to match their default color scheme on our "Create Post" button. This color code was saved in a SASS variable called $brand-primary. We also added two special styles for when the user hovers on the button (darken the text) and clicks down on it (darken background). Note the use of SASS's & selector to indicate the immediate parent element:

    // Bootstrap Colors
$brand-primary: #428bca;
...
.highlighted-btn{
  background-color: $brand-primary;
  a{
    color: white;
    &:hover{
      color: #eee;
    }
  }
  &:active{
    background-color: #085394;
  }
}

  6. The dropdown menu was basically copied from Bootstrap's docs and pared down to fit our needs:

    ...
<li class="dropdown">

    .nav-links > li{
  width: 195px;

5. Building the Search Form

The search form was much simpler to enact than the navbar. Bootstrap's custom styles really made things easy here.

Search Form Markup

  1. We started by wrapping everything in a new row div and applying some custom Bootstrap classes to narrow down the search wrapper to an 8-unit column.

  2. We set the inputs each inside their own columns so they could comfortably expand to fill those specific widths. The inputs were each also specified as "large" versions for bigger text and element height:

    ...
<div class="row">
  <section id="search-container" class="col-md-8 col-md-offset-2">
    <form>
      <div class="col-md-9">
        <input type="text" class="form-control input-lg" placeholder="Enter your search terms here...">
      </div>
      <div class="col-md-3">
        <input type="submit" class="btn btn-default btn-lg btn-block" value="Search">
      </div>
    </form>
  </section>
</div>
...

Search Form CSS

The CSS is quite simple since most of the heavy lifting has been done with Bootstrap's classes. All we need to do is tweak some of the spacings between elements:

#search-container{
  margin-top: 30px;
  margin-bottom: 10px;
  form{
    margin:auto;
  }
}

6. Building the Posts

Finally, we'll set up the posts. Most of the markup is recycled from the non-Bootstrap site since there isn't anything particularly flashy going on here (no widgets, dropdowns or buttons).

  1. We started by wrapping the whole posts section in a <section>, wrapping each post in an <article> which starts a new row, and finally nesting the interior content within a narrower column (remember that columns within columns are always based on a total of 12, so the inner col-md-9 is really 75% the width of the enclosing col-md-8):

    ...
<section id="posts" class="col-md-8 col-offset-2">
  <article class="row post">
    <div class="col-md-9 col-centered">
      ...

  2. We created a special class in order to actually center the columns because that's one thing Bootstrap doesn't to terribly well. If you have an even number of columns, you can easily do an offset like <div class="col-md-8 col-md-offset-2> but that doesn't work for odd numbers of columns like our col-md-9 so we need to hack together a solution on class we'll call col-centered:

    ...
.col-centered{
  float: none;
  margin: 0 auto;
}
...

  3. Finally, we more or less imported the markup from the previous file to handle all the text sizing and spacing, though we had to juggle a few extra changes around the <time> elements.

Comparing Bootstrap and "Naked" CSS Versions

Now you've seen how we built up the Bootstrap version of our page by essentially wrapping our old version in Bootstrap's special grid and widget classes. The aggregate amount of work it took to build this was roughly the same as before, but we get responsiveness for free and can more easily extend it by using other nifty components.

Bootstrap Pros and Cons

You've had a chance to see Bootstrap in action and it should be clear that it comes with its benefits and drawbacks. One obvious one follows what you just saw -- that Bootstrap has its own way of styling things and, as long as you're okay working with it, you can produce some excellent sites.

To highlight a few of the other items of note:

Pros

  • Using Bootstrap classes for alignment keeps your CSS cleaner of nitty-gritty things like custom rounding corners and manually creating grids.
  • Simple widgets are available to use immediately -- you saw the navbar dropdown working without requiring us to write our own custom JS or jQuery. The naked CSS site had a non-functional dropdown.
  • Try resizing the browser. For the most part, elements on the page degrade gracefully. If you're creating a real website, setting up your grid and its elements to degrade smoothly using media queries can frankly be a real pain. Bootstrap gives you mobile friendliness for free.
  • Once you figure out how they want their classes organized, creating with Bootstrap is fast, especially widgets that would otherwise take you a long time to perfect.
  • There is a consistent look and feel to the page -- Twitter's designers spent a long time perfecting the look of Bootstrap so it looks great out of the box.

Cons

  • Unfortunately, you can see that the HTML markup got a lot bigger. Adding all of bootstrap's classes caused us to nest deeper into our HTML, which makes it a bit more brittle. Sometimes, though, this can actually be a pro because it forces you to stay organized.
  • Bootstrap classes aren't quite semantic. We've talked a lot about how you should name your classes based on what they represent and push off all your styles to the CSS. Bootstrap bends and breaks many of those rules with classes like their grid sizing, floating, and other styling operations that get tacked onto the markup. It's not ideal because you can end up needing to change the markup to change the look of the page.
  • The biggest con? It's a pain in the ass to fight Bootstrap. If you want to make your navbar a specific height, you need to dig into the CSS and make sure you've overridden the right elements (which often includes children). Small-seeming changes can quickly become battles, especially if you're trying to preserve mobile-friendliness (sometimes it's easy to change something in the browser which breaks the mobile side).

Takeaways

The last point is very important -- as a beginner, bootstrap allows you to whip up a good-looking page quite fast (once you know how it works) but, if you're looking to turn a mockup into a pixel-perfect page, you'll quickly find yourself frustrated with it.

So... as a beginner, don't fight bootstrap if you don't have to!. If it's not pixel perfect, don't worry for now. Just go with what they give you. Follow their examples. Once you've skilled up to the point where you're more comfortable with CSS, then you can work to customize your Bootstrap-based code to create some really awesome stuff.
