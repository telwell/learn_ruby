Developer Tools and Tricks

This very brief lesson reviews some tools and tricks that will help you when you're building front-end pages.

Browser Developer Tools

Hopefully you already use the developer tools console in your browser (or how did you get this far...?), but do you know its secrets? We'll work with Google's Chrome browser here but there are similar tricks in Mozilla's Firefox browser and others as well.

Not Caching CSS

Can't figure out why your local changes aren't showing up in the browser? The stylesheet is probably being cached (check on the network tab to see if it's getting a "304 -- not modified" response code). Browsers do this because, if they see that they've already loaded a file once and it hasn't been updated, there's no reason to go grab it from the server. Unfortunately, they aren't always that good at figuring out whether the file they currently have cached is out of date or not.

If you don't want your stylesheet to be cached by the browser, disable caching while developer tools are open (under settings).

See this Stack Overflow Response for the detailed procedure.

Editing CSS Directly in the Dev Tools

If you find yourself frequently jumping into your text editor, making a minor CSS change, then refreshing your browser, it's probably easier to just edit the CSS directly in your dev tools Style pane until you find the values you want and then plug those into your text editor.

Trying it out yourself is the best way to learn. You can also check out Editing Styles from the Chrome team if you want to learn more.

Editing HTML Directly in the Dev Tools

You can also add new HTML tags or edit existing ones (e.g. to add classes or attributes) in the Elements pane. Either click directly into an existing element or add new ones by right clicking and choosing "Edit as HTML":

Triggering CSS Element States (e.g. Hover)

You frequently want to be able to see what an element would look like in its other states (e.g. :hover or :visited), but these won't naturally show up in your developer tools (even if you're performing the action that would lead to that state). You can manually trigger these states by clicking the little checkered-box icon in the top right of your style panel and then selecting which states to activate:

Browser Add-ins

Here are a few handy browser add-ins to help you when you're developing your front ends.

Pesticide

Pesticide will wrap every element on the page in a colored <div> so you can easily see how they are laid out. Highly recommended while developing or if you need to fix bugs!

WhatFont

Whatfont simply tells you what the font stack, size, color and origin of a given font is. Just click on any text once you've activated the add-on and it'll show an info box. This is helpful both to quickly debug your layouts (if you want to know your font stack without digging into the dev tools) and to check out the fonts of other websites.

960 Gridder

This helpful bookmarklet overlays any site with a 960 grid so you can see how it breaks down. Mostly useful for deconstructing other sites, we covered this back in the lesson on Grid Systems.

Sublime Text Tips and Shortcuts

You and Sublime will get to know each other well over the next several months. Knowing how to effectively use it will save you time and make you a happier programmer. In particular, get familiar with the following things.

Shortcuts

  • CMD + P to find and open a file
  • CMD + SHIFT + F to initiate a global search
  • CMD + click to select multiple lines at once... awesome.
  • Select text then use CMD + D to ALSO select the next instance of this text. This is awesome**2.
  • CMD + / to comment out all selected lines.
  • Select multiple lines and use TAB or SHIFT + TAB to adjust their indentation all at once.
  • CMD + L to select the entire line
  • CMD + 1,2,3... to jump to that numbered tab

Multiple Columns

View >> Layout >> "Columns: 2". One for HTML, one for CSS... you're welcome.

Code Snippets

Look into code snippets -- you can essentially just start typing something like "div.some-class" and then hit tab or enter to fill out the full div structure automatically to <div class="some-class"></div>.

It also intelligently tabs you from within the element to outside of it. You can create any of these awesome shortcuts you want -- anything you find yourself doing constantly can be made into a shortcut.

See the Resources tab for more info.

Packages

Finally, packages add useful bits of functionality to the editor. They function much like Chrome Add-ons, though there's a bit of extra complexity in the set-up.

If you're editing Markdown frequently, look into the markdown editing packages. There are also other syntax highlighting packages and "linting" packages which look for errors in your code in real time.

Package Control can be installed from here.

To install a Sublime "package" with PackageControl:

  1. CMD+SHIFT+P
  2. Start typing "pack inst", and from the list choose "Package Control: Install Package".
  3. Wait a moment and it will list non-installed packages. Select the one you want, press Enter and wait a few seconds. (Press Esc or click away to cancel.)

A couple of useful packages:

  1. SublimeLinter, a self-contained "linter", to point out syntactic errors and style suggestions next to the line numbers.
  2. TrailingSpaces, which highlights and/or deletes trailing spaces to keep your code clean.
