Demo: From Pseudocode to Ruby Code

You've had some experience from the previous mini-course with writing pseudocode to show a procedure or express your solution to a problem. In this quick demo, we just want to show you how quickly pseudocode becomes Ruby code. You don't need to know any Ruby yet of course -- we just want to give you a taste for what's ahead.

The Deaf Grandma

In one of the resources that you'll see in the coming lessons (Chris Pine's Learn to Program), there is a problem called the "Deaf Grandma". In the problem, you are talking to an old grandma who can't really hear to well. The problem is laid out as follows:

  • No matter what you say to the grandma, she says "HUH!? SPEAK UP, SONNY!"
  • If you "shout" your input (e.g. typing all capital letters), she yells back "NO, NOT SINCE 1938!"
  • The only way to leave is if you shout "BYE".

This sounds like a good pseudocoding exercise, right? It might look something like what we have below:

# PSEUDOCODE
PROGRAM DeafGrandma
    Keep LOOPING to get more input

        Ask what to say to grandma

        IF we said "BYE"
            stop asking for more input and leave
        END

        IF what I just said is too quiet
            Grandma responds "HUH!? SPEAK UP, SONNY!"
        ELSE
            Grandma responds "NO, NOT SINCE 1938!"
        END

    END
END

Great! Now what might this look like in Ruby?

There are only a few differences:

  1. We'll need to wrap the whole thing in a Ruby-style method and use the right loop so the program continues onward until we're ready to stop
  2. We'll have to get some user input by asking the user to type something into the command line and keep track of it somehow
  3. We'll need to convert the pseudo logic for Grandma's responses to Ruby syntax
  4. We'll need to express our logic for leaving ("BYE!") in Ruby as well.

Let's just start at the top of that list by setting up a Ruby method and our while loop which keeps asking for input continuously until we tell it to stop. We'll use our Pseudocode as comments:

def deaf_grandma

    # Keep LOOPING to get more input
    while true

        # IF we said "BYE"
            # stop asking for more input and leave
        # END

        # Ask the user what to say to grandma

        # IF what I just said is too quiet
            # Grandma responds "HUH!? SPEAK UP, SONNY!"
        # ELSE
            # Grandma responds "NO, NOT SINCE 1938!"
        # END
    end
end

Next, let's actually get that user input and store it in a variable:

def deaf_grandma

    # Keep LOOPING to get more input
    while true

        # Ask the user what to say to grandma
        puts "What do you want to say to Grandma?"
        tell_grandma = gets.chomp

        # IF we said "BYE"
            # stop asking for more input and leave
        # END

        # IF what I just said is too quiet
            # Grandma responds "HUH!? SPEAK UP, SONNY!"
        # ELSE
            # Grandma responds "NO, NOT SINCE 1938!"
        # END

    end

end

Now we can change our grandma's response pseudo-logic into Ruby logic:

def deaf_grandma

    # Keep LOOPING to get more input
    while true

        # Ask the user what to say to grandma
        puts "What do you want to say to Grandma?"
        tell_grandma = gets.chomp

        # IF we said "BYE"
            # stop asking for more input and leave
        # END

        # IF what I just said is too quiet
        if tell_grandma.upcase != tell_grandma
            # Grandma responds "HUH!? SPEAK UP, SONNY!"
            puts "HUH!? SPEAK UP, SONNY!"
        # ELSE
        else
            # Grandma responds "NO, NOT SINCE 1938!"
            puts "NO, NOT SINCE 1938!"
        # END
        end

    end

end

And, finally, the last step is to finish up with the logic for actually leaving the program if we've said "BYE":

def deaf_grandma

    # Keep LOOPING to get more input
    while true

        # Ask the user what to say to grandma
        puts "What do you want to say to Grandma?"
        tell_grandma = gets.chomp

        # IF we said "BYE"
        if tell_grandma == "BYE"
            # stop asking for more input and leave
            break
        # END
        end

        # IF what I just said is too quiet
        if tell_grandma.upcase != tell_grandma
            # Grandma responds "HUH!? SPEAK UP, SONNY!"
            puts "HUH!? SPEAK UP, SONNY!"
        # ELSE
        else
            # Grandma responds "NO, NOT SINCE 1938!"
            puts "NO, NOT SINCE 1938!"
        # END
        end     

    end 

end

That's it! If we remove all the extra comments, you'll see how the pseudocode looks side-by-side with the Ruby code:

Our Pseudocode

# PSEUDOCODE
PROGRAM DeafGrandma
    Keep LOOPING to get more input

        Ask what to say to grandma

        IF we said "BYE"
            stop asking for more input and leave
        END

        IF what I just said is too quiet
            Grandma responds "HUH!? SPEAK UP, SONNY!"
        ELSE
            Grandma responds "NO, NOT SINCE 1938!"
        END

    END
END

Our Ruby Code

# RUBY CODE
def deaf_grandma
    while true

        puts "What do you want to say to Grandma?"
        tell_grandma = gets.chomp

        if tell_grandma == "BYE"
            break
        end

        if tell_grandma.upcase != tell_grandma
            puts "HUH!? SPEAK UP, SONNY!"
        else
            puts "NO, NOT SINCE 1938!"
        end     
    end 
end

Try running this in your IRB (type irb on the command line). You'll need Ruby installed for this. You do this by pasting the script above into IRB and then running it by typing simply > deaf_grandma (note: the carrot > is not actually included -- that just shows that you're typing it at the IRB prompt).

When it asks for input, just type your phrase and hit ENTER. If you get stuck in a loop somehow and can't leave, use CTRL + C to break the loop.

The Punchline

Hopefully you got a sense for how easy it is to convert pseudocode into Ruby code. You may have no clue what exactly each method does but it should be pretty obvious that Ruby follows a very English-like syntax and the logic of your pseudocode translates directly to it.

This should give you faith in the power of pseudocode as a good first step towards solving a problem and some curiosity for how to do it yourself by learning Ruby. Luckily, that's exactly what we'll cover in the next few lessons...
