Review of Concepts

Congratulations on getting up to speed on markup with HTML/CSS and coding with Ruby! There's a lot of stuff packed into these lessons and you should be feeling pretty good about where you've gotten. Let's review some of the things we covered. This is just meant to be a high level refresher.

Web Markup with HTML/CSS

Webpages are put together using the HTML markup language and styled using Cascading Style Sheets. HTML tags have an open and a closing tag and wrap content on your page to tell the browser what it is (and how therefore to display it).

HTML tags can be identified using classes (which can apply to many tags) or IDs (for unique tags), which allows you to target them with your CSS code. Some common HTML tags:

# Generic wrappers:
<div>: A generic wrapper element, commonly used to 
        group items together for CSS styling.  
        Block level. (use <span> for inline)
<span>: An inline wrapper for elements you want styled 
        a particular way.
<body>: The main content wrapper tag of your document
<html>: The main wrapper tag for your HTML page -- 
        includes <head> and <body> within
<head>: The stuff at the top of the page that the 
        browser uses but you can't see

# Elements for the document's <head> tag
<meta>: Metadata for your document (in the HEAD tag)
<title>: a <head> element that defines the title of 
        the document
<style>: Wraps CSS styles. Anything inside is treated
        as CSS.
<link>: Links to import other documents in the HEAD tag 
        like stylesheets.

# Inline Wrappers for Content
<a>: Hyperlink.  You must specify the href attribute.
<b>: Bold text.  Usually better to just specify it 
        with CSS and a <span>.
<code>: Wraps code snippets so they get displayed 
        differently... like this section!
<em>: Emphasis (italic).  Usually better to just 
        specify it with CSS and a <span>.
<strong>: Important text (bolded usually).

# Block-level elements
<h1>,<h2> etc: Headings, ordered by hierarchy
<br>: A line break.  Don't use this -- make spacing 
        with CSS instead.
<hr>: Divider line
<img>: Defines an image pointed to by the src attribute.
<ol>: Wraps an ordered (numbered) list
<ul>: Wraps an UNordered (bulleted) list
<li>: An item inside a list.
<table>, <tbody>,<tr>,<td>,<th>: Table wrapper and table 
        contents.  Infrequently used.

# Forms
<form>: Wraps the input fields of a form so the browser 
        knows what to include when it's submitted.
<button>: Clickable buttons, often wrapped in anchor 
        tags
<input>: An input field for a form
<optgroup>,<option>: Specifies the contents of a 
        dropdown list of choices
<select>: Defines a dropdown list
<textarea>: A larger input field for inputting text.

Block-level elements take up the entire line on the page but they can use the CSS box model for their stylings. Inline elements don't take up any extra space on the page and usually just modify the words within them.

With CSS, targeting properly is the name of the game. You can target specific tags, attributes of tags, classes, IDs and more. You can target specific elements by nesting selectors -- e.g. you can ask for the list item inside the element with the "product-item" class inside the element with the "inventory-listing" ID.

Style sheets "cascade" because the styles that apply to a parent element also apply to its children unless they are overridden. That helps us to avoid constantly repeating ourselves.

CSS allows you to do basically anything to elements on the page -- resizing, coloring, hiding, displaying different ways etc. You can define the styles of an element itself or the relationships between elements on the page (e.g. their spacing relative to each other).

CSS units are specified in terms of absolute pixels (px), "ems" which are relative to the font size of the parent element, "rems" which are relative to the document's base font size, and % which acts like "ems".

Some common CSS styles:

# Styling Text
color: sets the text color (not the background 
        color!)
text-align: choose how to align text inside a block
        element, e.g. center, left, right, justify
text-decoration: can be set to add an underline
line-height: the total height taken up by a line of 
        text
letter-spacing: the additional space between each 
        letter

# Styling Fonts
font-family: The prioritized list of fonts for the
        browser to try using, e.g. Helvetica, serif
font-size: text size in ems, rems, or px
font-style: used to add italics
font-weight: used to determine bold or skinny fonts

# Background
background-color: color of the Background
background-image: adds an image to the background
background-repeat: determines how the image repeats
background-position: determines the position of BG image

# Lists
list-style-type: what the bullets or numbers look like

# Box-Model / Positioning
margin: the minimum distance between an element and
        its nearest neighbor.
padding: the buffer zone inside the element's box 
        between its border and its content
border: The type, color, and sizing of the border
float: whether and how a box should float on the page
clear: specifies where no floating is allowed
position: choose if an element is fixed in place,
        bumped relative to its current position, or
        absolutely positioned relative to its nearest
        eligible parent element.
display: used to remove elements from the page

CSS styles can be applied directly to an element in the HTML, added between <style> tags, or imported from a separate stylesheet.

HTML5 and CSS3 have introduced more types of elements, interactions and effects but you don't need to worry about them while you're just getting started.

Web Programming with Ruby

Ruby is a the high level programming language designed for programmer happiness. It uses a very sparse syntax to make your life easier and is the core language used by Ruby on Rails. It's very easy to turn the pseudocode from your solution to a problem into Ruby code once you've picked up its syntax.

Everything in Ruby is an object. That means you can pass them around and ask questions of them pretty much at will. Once you've mastered thinking in an object-oriented fashion, it starts to really make sense.

The basic building blocks of Ruby are the same as any other object-oriented programming language -- strings, arrays and hashes store information while scripts are contained within methods which are contained in larger modules called classes. These classes allow you to "construct" new objects with the #new method.

Ruby uses conditional operators like "if" statements to direct control flow and contains a lot of iterators which are useful for different kinds of scenarios. Many of these operate on data structures like arrays and take an argument called a "block", which is basically just an anonymous function to run with each iteration.

As a beginner working with Ruby, try to focus on keeping your problem solutions as simple as possible and let the computer do the work (ie. "brute force" it).
